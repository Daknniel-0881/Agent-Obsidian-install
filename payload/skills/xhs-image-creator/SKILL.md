# 小红书作图 Skill

小红书图文内容生产流程。Bento Grid 视觉风格 + 口语写作语气 + 飞书云文档交付。

## 触发词

- 小红书作图
- XHS图片 / XHS images
- 做小红书图
- 制作小红书内容图

## 快速用法

```
/xhs-image-creator [内容来源]

# 示例
/xhs-image-creator https://example.com/article    # 从URL抓取
/xhs-image-creator article.md                     # 本地文件
/xhs-image-creator                                # 直接粘贴内容
```

## 视觉规范（Bento Grid Style）

所有图片必须严格遵循以下规范，不允许自行切换风格。

### 布局
- Bento Box便当盒式网格，每张图划分 **5~7个区域**
- 区域之间保留明显间隙（gap），避免死板，可适当打破边界
- 可参考思维导图式的放射结构

### 背景与留白
- 纯净米白色或羊皮纸色背景
- 留白率（Negative Space）**至少20%**，给眼睛呼吸空间

### 元素密度
- 每个区域内只放 **1~2个** 主要图标或插画
- 线条：手绘风箭头、毛边、装饰线

### 配色原则（10-30-60）
- **60%** 背景色（米白/羊皮纸）
- **30%** 中性衔接色（浅灰/象牙白/暖米）
- **10%** 点缀色作为唯一视觉高光（珊瑚色/橙色/青色，每张图保持一致）

### 整体质感
- 扁平化设计（Flat Design）+ 微带一点点投影增加层次感
- 风格参考：Notion风格插画、Apple Keynote发布会风格、瑞士平面设计

### 图片规格
- 尺寸：**1242×1660px**（3:4竖版）
- 格式：PNG原图，不压缩
- 总数：通常 **9张**（可根据内容调整为7~10张）
- 模型：`google/gemini-3.1-flash-image-preview`（通过OpenRouter）

## 统一Prompt模板

每张图的prompt由两部分拼接：

### 风格前缀（固定，每张图都加）

```
Create a vertical 3:4 ratio illustration in Bento Grid layout style. Cream/parchment background, flat design, hand-drawn arrows, 60-30-10 color rule (cream/neutral/coral accent). Notion + Apple Keynote + Swiss design style. 5-7 grid areas with visible gaps. 20% negative space minimum. Micro shadows for depth. PNG 1242x1660px.
```

### 内容描述（每页单独写）

描述该页的：标题文字、区域划分方式、每个区域放什么图标/插画、关键数据或文案。

## 写作风格规范

### 核心三字诀
**有趣、有思考、有判断** — 不是信息搬运，必须带用户立场。

### 语气要求
- 口语化，像朋友聊天，不像AI写报告
- 结论先行，再展开论据
- 永远具体：不说"很多客户"，说"80%的客户"
- 问号 >> 感叹号

### 情绪控制
- 不用"放大招""炸裂""颠覆"这类一惊一乍的词
- 兴奋靠"真的""确实"加重，不靠喊
- 适度暴露不确定："大概率""说不准""也许吧"

### 标题规范
- 直截了当，不堆砌修饰语
- 封面标题可分两行，像说话一样自然
- 好例子：「Anthropic给全球企业 / 做了一套AI数字员工系统」
- 坏例子：「Claude放大招：AI员工上线即干活」

### 禁用词（写完必查）
赋能、链路、飞轮、认知升级、颗粒度、抓手、闭环、触达、矩阵、范式、痛点、底层逻辑（单独用）、护城河（泛用）、降维打击、生态位、心智模型、破折号（——）

### 禁用句式
「不是…而是…」「不是…是…」「并非…而是…」— AI味极浓，改用直接陈述或用"说白了""本质上"引出判断

### 字数限制
- **正文不超过1000字**（小红书平台硬限制）
- 写完必须数字数，超了就砍，优先砍重复论据和过渡句

## 完整作图流程

### 进度追踪

```
XHS作图进度：
- [ ] Step 1：资料收集与翻译
- [ ] Step 2：确认标题和副标题
- [ ] Step 3：规划9页内容结构（先写规划，不急于生图）
- [ ] Step 4：发飞书云文档，等用户确认规划
- [ ] Step 5：逐张生图（调用 generate-image skill）
- [ ] Step 6：原图插入飞书云文档交付
- [ ] Step 7：按反馈迭代单张图
```

---

### Step 1：资料收集

**输入类型**：
- URL → 抓取全文（参考 scraping-rules.md + GENOME.jsonl路由表）
- 文件 → 直接读取
- 粘贴内容 → 保存为 `source.md`

**处理**：
1. 保存原文（英文内容同时提供中文翻译摘要）
2. 提炼核心观点：主题、关键数据、亮点案例
3. 识别目标读者（小红书用户画像）
4. 判断内容类型：干货科普 / 产品测评 / 观点输出 / 教程指南

---

### Step 2：确认标题和副标题

提供 **3组候选标题** 供用户选择：

```
封面标题候选：

1. [标题行1]
   [标题行2（可选）]
   副标题：[副标题]

2. ...

3. ...
```

**标题原则**：
- 结论先行，直击核心
- 口语化，像朋友聊天
- 避免营销号风格（禁用放大招/炸裂/颠覆）
- 中国读者熟悉的比喻优先（避免宝可梦模拟器这类西方比喻）

等用户确认后再进行下一步。

---

### Step 3：规划9页内容结构

**先规划，不生图。** 每页输出以下结构：

```markdown
## P1 封面
- 核心信息：[标题文字]
- Bento区域规划：[描述5-7个区域的内容安排]
- 文案语气：[这页要传递的情绪/立场]
- 关键视觉元素：[图标/插画建议]

## P2 [页面主题]
...
```

**通用9页结构（可根据内容调整）**：

| 页码 | 类型 | 作用 |
|------|------|------|
| P1 | 封面 | Hook + 视觉吸引，确认读者往下刷 |
| P2 | 背景/问题 | 建立认知基础，为什么这件事重要 |
| P3 | 核心观点1 | 第一个关键信息 |
| P4 | 核心观点2 | 第二个关键信息 |
| P5 | 核心观点3 | 第三个关键信息 |
| P6 | 数据/案例 | 支撑论据，增加可信度 |
| P7 | 对比/分析 | 深化理解，给判断 |
| P8 | 实操/结论 | 给读者可操作的takeaway |
| P9 | 结尾 | 总结 + 引导互动（收藏/评论/关注） |

---

### Step 4：规划确认（飞书云文档）

将内容规划写入飞书云文档，以五竹（bot）身份发给用户：

```
标题：[主题] 小红书图文内容规划

内容：
- 标题确认：[选定标题]
- 9页结构规划：[Step 3的完整规划]
- 待确认事项：[如有疑问列出]
```

**等用户确认后再开始生图。**

---

### Step 5：逐张生图

调用 `/generate-image` skill，**按顺序逐张生成**。

**每张图的操作**：

1. 拼接prompt：
   ```
   [风格前缀] + [该页内容描述]
   ```

2. 内容描述写法示例（P1封面）：
   ```
   Page 1 cover. Title text: "[主标题]" large in center-top area. 
   Subtitle: "[副标题]" below. 
   Bento areas: top-left [图标描述], top-right [图标描述], 
   center [核心视觉], bottom-left [数据/文字], bottom-right [装饰].
   Accent color: coral/orange.
   ```

3. 文化适配检查（生成prompt时自动应用）：
   - 比喻用中国读者熟悉的（Type-C线 > 宝可梦模拟器）
   - 技术概念翻译成大白话
   - 有公司/产品案例时加对应logo提示
   - 数据标注来源

4. 生成后检查：视觉是否符合Bento Grid规范，文字是否清晰

**生成失败处理**：重试一次，调整prompt描述后再试。

---

### Step 6：原图交付飞书云文档

1. 将9张PNG原图（无压缩）插入飞书云文档
2. 图片按顺序排列，每张图下方标注页码和主题
3. 同时提供两份文档：
   - **内容规划文档**：9页结构规划 + 文案
   - **成品图文档**：9张原图排列
4. 以五竹（bot）身份通知用户："[主题]小红书图文已完成，请查收"

---

### Step 7：迭代修改

按用户反馈重新生成单张图：

1. 定位需修改的页码（如P3）
2. 调整该页的内容描述
3. 重新生成单张，替换原图
4. 更新飞书云文档对应位置

**不需要重新生成全套**，单张替换即可。

## 知识库调用（必读）

**小红书内容知识库路径**：`$VAULT_ROOT/Areas/内容创作/小红书/`

```
小红书/
├── 发布案例/          # 实际发布的标题+正文原文（每篇一个md）
├── 风格拆解/          # 基于真实案例的风格分析
│   └── 小红书写作风格拆解.md   # 核心风格文档
└── （后续可扩展：选题库/、数据复盘/）
```

### 写文案前必须做的事

1. **先读风格拆解**：`Grep` 搜索 `风格拆解/小红书写作风格拆解.md`，加载标题规则、开头钩子、解释手法、节奏结构、情绪表达、对标检查清单
2. **再看发布案例**：`Glob` 扫描 `发布案例/*.md`，读取最近3篇，感受真实语感
3. **写完对照检查**：用风格拆解末尾的「对标检查清单」逐条过

### 关键风格要点速查（详见风格拆解文档）

- **标题**：一句话说清"谁做了什么" + 末尾克制emoji（😨😅🤔），不用感叹号
- **开头**：第一句就有判断/钩子（"完了，""说实话，"），不铺垫
- **翻译**：每个技术概念配"说白了""相当于"翻译成大白话
- **比喻**：中国意象优先（战马缰绳、Type-C线），不用西方梗
- **编号**：用 1️⃣2️⃣3️⃣ emoji，不用 markdown 列表
- **节奏**：短段为主（≤4行），长段之间插短句呼吸（"天才。。"）
- **收尾**：极短留白（2~4字），不总结不升华
- **省略号**：用".."替代"……"
- **人称**：解释用"你"代入，推演用"那我"模拟决策

### 发布后沉淀

每次用户发布新的小红书内容后，立即：
1. 将标题+正文原文保存到 `发布案例/YYYY-MM-DD-主题.md`
2. 如果出现新的风格特征，追加到 `风格拆解/小红书写作风格拆解.md`
3. 持续校准风格模型，案例越多越准

## 注意事项

- 本skill是**流程指导skill**，不包含独立脚本，生图依赖 `generate-image` skill
- 规划确认前不生图，避免返工
- 原图PNG不压缩，保持最高质量
- 每次对话开始，先确认用户提供的内容来源（URL/文件/粘贴）
- **写文案/标题前必须先读知识库**，不凭记忆写

## 可选风格预设（Style Presets）

除默认 Bento Grid 风格外，本 skill 维护一个 `style-presets/` 目录，存放经过反向拆解+跨主题验证的备选风格 prompt。用户指定走某个预设时，用预设文件里的"风格段"替换默认 Bento 风格前缀，画面段按主题特化。

### 当前可用预设

| 预设名 | 文件 | 适用场景 | 来源 |
|--------|------|---------|------|
| **stark-nico99 工程速写（横版 16:9）** | `style-presets/stark-nico99-engineering-sketch-horizontal.md` | 课程笔记 / 知识科普 / 流程拆解，需要"手绘工程速写"质感的横版图 | 反向拆解 @stark_nico99，V11 浅色虚线主导版 |
| **stark-nico99 工程速写（竖版 3:4）** | `style-presets/stark-nico99-engineering-sketch-vertical.md` | 小红书规格（1242×1660），同上风格的竖版排布 | 同上，TOP→BOTTOM 三块叠加版 |

### stark-nico99 工程速写风格速记

- 纯白底 ≥35% 留白，**虚线为主，实线只用于图标外轮廓 + 标题下划线**
- 水彩透明度 15-25%（非常淡，不是彩色平涂），调色板：浅灰蓝 / 浅青豆绿 / 浅杏色 / 浅奶油黄
- 石墨灰铅笔线 #3A3A3A（不要纯黑）
- 螺旋每圈大小不一致、手抖感（**显式禁词：NEVER vector spring or geometric coil**）
- 时间轴箭头尾端水彩晕染（**禁词：NEVER clean solid triangular arrowhead**）
- 虚线圆角框 ≤2 处，只给数据指标卡 / 宣言列表用
- 图标 = 迷你工程插画 ≥3 个内部结构元素（不是简笔外壳）
- 中文照原文渲染，不翻译

### 调用示例

```
# 横版课程速写
"按 xhs-image-creator 的 stark-nico99-engineering-sketch-horizontal 预设生图，主题：[xxx]"

# 竖版小红书图
"按 xhs-image-creator 的 stark-nico99-engineering-sketch-vertical 预设生图，主题：[xxx]"
```
