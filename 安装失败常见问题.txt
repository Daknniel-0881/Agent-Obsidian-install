排查流程：从问题 1 开始往下找，命中哪条按哪条做。
每条都搞不定，最底部有【收集日志】方法，发给管理员。

───────────────────────────────────────────────────────────────
【问题 1】脚本提示"找不到安装包 / Cannot find installer"
───────────────────────────────────────────────────────────────

  原因：脚本在以下 4 个目录都没找到 .dmg/.exe/.deb 文件：
    1. ~/Downloads/自动部署脚本/<系统>/
    2. ~/Desktop/自动部署脚本/<系统>/
    3. C:\自动部署脚本\<系统>\（Windows 专属）
    4. D:\自动部署脚本\<系统>\（Windows 专属）

  解决：
    方案 A（推荐）：把【自动部署脚本】整个文件夹移动到
                    Downloads 或 桌面（保留中文名不变）
    方案 B：终端运行时显式指定路径：
            Mac/Linux: bash install-mac.sh --pkg-dir /your/path
            Windows:   .\install-windows.ps1 -PkgDir D:\your\path

───────────────────────────────────────────────────────────────
【问题 2】Mac 提示"无法验证开发者 / 已阻止使用"
───────────────────────────────────────────────────────────────

  这是 Apple 官方安全机制（CodePilot 暂未在 App Store 签名）。
  解决：
    ① 系统设置 → 隐私与安全性 → 滑到底部
    ② 看到"已阻止 CodePilot.app" → 点【仍要打开】
    ③ 输入开机密码 → CodePilot 启动

  脚本会自动 xattr -dr 移除隔离属性，多数情况无需此步。
  极少数 macOS 14+ 仍会拦截，按上面步骤手动放行即可。

───────────────────────────────────────────────────────────────
【问题 3】Windows 提示"无法加载文件，因为系统禁止运行脚本"
───────────────────────────────────────────────────────────────

  原因：PowerShell 默认禁止运行未签名脚本。
  解决：
    ① 在开始菜单搜"PowerShell" → 右键 → 以管理员身份运行
    ② 在 PowerShell 里运行：
       Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser -Force
    ③ 关闭 PowerShell，重新双击 install-windows.bat

───────────────────────────────────────────────────────────────
【问题 4】网络超时 / GitHub 访问失败
───────────────────────────────────────────────────────────────

  解决：
    ① 脚本会自动检测网络并切换到 USTC 镜像，多数自动恢复
    ② 仍失败：开 VPN 或切到 4G/5G 热点重试
    ③ 完全无网络：使用「离线包扩展版」（联系管理员获取）

───────────────────────────────────────────────────────────────
【问题 5】npm 安装 @anthropic-ai/claude-code 卡住
───────────────────────────────────────────────────────────────

  解决：
    ① 终端运行：npm config set registry https://registry.npmmirror.com
    ② 重试：npm install -g @anthropic-ai/claude-code
    ③ 仍失败：rm -rf ~/.npm 后重试

───────────────────────────────────────────────────────────────
【问题 6】Obsidian 命令行界面（CLI）找不到 obsidian 命令
───────────────────────────────────────────────────────────────

  原因：Obsidian 高级设置中"命令行界面"未开启。
  手动开启步骤：
    ① 打开 Obsidian → 左下角齿轮图标（设置）
    ② 点【关于】→ 滑到底部【高级】区域
    ③ 找到「命令行界面」开关 → 打开
    ④ 弹出窗口点【注册】→ 完成
  开启后：
    Mac/Linux：在终端运行 which obsidian，应能找到路径
    Windows：在 PowerShell 运行 Get-Command obsidian

───────────────────────────────────────────────────────────────
【问题 7】CodePilot 客户端打开后没有 API Key 输入框
───────────────────────────────────────────────────────────────

  解决：
    ① 客户端左下角点【设置】图标（齿轮状）
    ② 选择【服务商】（Anthropic / OpenRouter / 其他）
    ③ 选择类型并粘贴你的 API Key
    ④ 保存后客户端自动连接

  没有 API Key？
    Anthropic：https://console.anthropic.com/settings/keys
    OpenRouter：https://openrouter.ai/keys

───────────────────────────────────────────────────────────────
【问题 8】飞书 Lark CLI 授权失败
───────────────────────────────────────────────────────────────

  原因：Lark CLI 需要在 OAuth 授权页登录企业账号。
  解决：
    ① 终端运行：lark auth login
    ② 自动跳转浏览器 → 用企业飞书账号登录
    ③ 授权 lark-cli 应用 → 终端显示"login success"
    ④ 浏览器没自动跳转：复制终端输出的 URL 手动打开

───────────────────────────────────────────────────────────────
【问题 9】脚本中途崩溃，想从断点继续
───────────────────────────────────────────────────────────────

  解决：
    ① 重新运行同一个入口脚本（.command/.bat/.sh）
    ② 已安装的组件会自动跳过（脚本有幂等检测）
    ③ 想完全重装：
       Mac/Linux：rm -rf ~/.claude && 重新运行
       Windows：rmdir /s %USERPROFILE%\.claude && 重新运行

───────────────────────────────────────────────────────────────
【问题 10】24 小时待机设置无效，电脑还是会休眠
───────────────────────────────────────────────────────────────

  Mac 解决：
    ① 系统设置 → 节能模式 → 勾选"防止显示器关闭时电脑进入睡眠"
    ② 或终端运行：sudo pmset -a disablesleep 1

  Windows 解决：
    ① 控制面板 → 电源选项 → 选择电源计划 → 更改计划设置
    ② "使计算机进入睡眠状态" 改为 "从不"

───────────────────────────────────────────────────────────────
【问题 11】离线包检测异常 / 想强制走在线 or 离线
───────────────────────────────────────────────────────────────

  脚本默认自动判断，特殊情况可手动指定：
    强制在线：bash install-mac.sh --online-only
              .\install-windows.ps1 -OnlineOnly
    强制离线：bash install-mac.sh --offline
              .\install-windows.ps1 -Offline

───────────────────────────────────────────────────────────────
【问题 12】Homebrew 安装失败（仅 Mac）
───────────────────────────────────────────────────────────────

  脚本会自动尝试官方源 → 清华源 → USTC 源。
  仍失败的话：
    ① 先手动装：
       /bin/bash -c "$(curl -fsSL https://gitee.com/cunkai/HomebrewCN/raw/master/Homebrew.sh)"
    ② 装完后重新运行脚本，会跳过 Homebrew 步骤

───────────────────────────────────────────────────────────────
【收集日志（找管理员前请提供）】
───────────────────────────────────────────────────────────────

  Mac/Linux：
    cat ~/.claude/logs/install-*.log | tail -200 > /tmp/install-error.txt
    把 /tmp/install-error.txt 发给管理员

  Windows（PowerShell）：
    Get-Content $env:USERPROFILE\.claude\logs\install-*.log -Tail 200 |
        Out-File $env:USERPROFILE\Desktop\install-error.txt
    把桌面的 install-error.txt 发给管理员

  发送时同时附带：
    - 你的操作系统版本（Mac 版本号 / Win 10 还是 11）
    - 终端最后几行红色错误信息（截图也行）
    - 你执行的命令（哪个脚本、有没有加参数）
